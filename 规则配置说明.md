# 站点专属规格提取规则说明

## 这个功能解决什么问题

不同 Shopify 独立站的商品参数区块不统一。默认规则会优先从 `Specification`、`Specs`、`Technical Specs`、`Details` 等文本附近提取规格内容，但有些站点会使用特殊标题、折叠组件或非标准页面结构。

遇到默认规则解析不准时，可以给某个站点配置一份专属 JSON 规则。规则只负责从落地页 HTML 中提取“准备交给 LLM 的规格文本”，不直接替代 LLM。

## 后续标准流程

1. 用户发现某个站点参数解析不准。
2. 用户把该商品落地页 URL 发给 Codex。
3. Codex 分析页面结构，生成一份站点专属规则 JSON。
4. 用户进入系统的“站点管理”。
5. 点击对应站点的“规则配置”。
6. 粘贴 JSON。
7. 选择一个商品点击“测试提取”。
8. 如果预览文本能稳定包含规格参数，就保存规则。
9. 后续该站点的“解析参数”和“刷新参数”都会自动使用这份规则。

## 当前支持的 JSON 字段

```json
{
  "keywords": ["specification", "specifications", "technical specs", "tech specs", "product details", "details"],
  "excludeKeywords": ["reviews", "you may also like", "related products", "shipping", "warranty", "returns"],
  "takeLines": 80,
  "fallbackLines": 220,
  "maxChars": 18000,
  "includeFallback": true
}
```

- `keywords`：命中这些标题或关键词后，从这里开始截取文本。
- `excludeKeywords`：截取过程中遇到这些词就提前停止，避免混入评论、推荐商品、售后政策。
- `takeLines`：每次命中关键词后最多截取多少行。
- `fallbackLines`：没有命中关键词时，兜底截取页面前多少行。
- `maxChars`：最终送给 LLM 的最大字符数。
- `includeFallback`：是否启用兜底截取。

## 给 Codex 的提醒话术

后续可以直接这样提醒：

```text
这是竞品分析平台的站点专属规格提取规则问题。请看项目里的 规则配置说明.md。
这个落地页默认解析不准：{落地页 URL}
请帮我生成一份可以粘贴到“站点管理 -> 规则配置”的 JSON，并说明我应该怎么测试。
```

## 注意

- 如果测试提取出来的文本已经包含完整规格，但 LLM 结果仍不准，应该优化 LLM 提示词或参数 schema。
- 如果测试提取出来的文本本身就不包含规格，应该调整站点专属规则。
- 规则保存后，只影响对应站点，不影响其他站点。
