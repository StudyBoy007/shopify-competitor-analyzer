# 分享给别人访问的方案

## 当前推荐

第一版使用：

```text
Cloudflare Tunnel + 应用访问密码
```

使用方式：

### 方式一：一键正式体验，推荐

```bash
npm run serve
```

这个命令会自动执行：

```text
npm run build
启动本地 API
启动前端 preview
启动 Cloudflare Tunnel
启动 caffeinate 防止电脑睡眠
```

终端会输出一个 `https://xxxx.trycloudflare.com` 链接。

把这个链接和访问密码发给使用者。

### 方式二：开发模式

开发时使用：

```bash
npm run dev
```

如果开发模式也想分享，再另开一个终端：

```bash
npm run share
```

## 关闭服务

无论是开发模式还是正式体验模式，都可以执行：

```bash
npm run stop
```

它会尝试关闭：

```text
5174 本地 API
5175 前端服务
Cloudflare Tunnel
caffeinate 防睡眠进程
```

访问密码保存在：

```text
.env.local
```

字段名：

```text
ACCESS_PASSWORD
```

## 安装 cloudflared

如果执行 `npm run share` 提示没有 `cloudflared`，macOS 可以安装：

```bash
brew install cloudflared
```

## 临时链接和固定域名

`npm run share` 使用的是 Cloudflare Quick Tunnel：

```text
优点：不用登录 Cloudflare，启动快
缺点：每次链接可能变化
```

如果要固定域名，例如：

```text
https://bike-compare.example.com
```

需要：

1. 有一个域名。
2. 域名接入 Cloudflare。
3. 在 Cloudflare Zero Trust 里创建 Tunnel。
4. 将子域名指向本机服务：

```text
http://localhost:5175
```

## 安全说明

- 不要直接暴露 `5174`，它是本地 API。
- 只分享 `5175` 对应的前端地址或 Cloudflare Tunnel 地址。
- 分享期间你的电脑、`npm run dev`、`npm run share` 都必须保持运行。
- 如果不再使用，关闭终端即可停止外部访问。
- 如果访问密码泄露，修改 `.env.local` 里的 `ACCESS_PASSWORD` 后重启 `npm run dev`。
